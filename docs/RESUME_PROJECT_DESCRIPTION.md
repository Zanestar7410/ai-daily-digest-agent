# Resume Project Description

## One-Line Version

Built an agent-augmented AI intelligence system that turns trusted-source updates into structured events, searchable historical records, and reusable research reports.

## Two-Bullet Version

- Designed a source-governed AI intelligence pipeline that discovers recent updates, extracts structured event metadata, and persists items, events, and research outputs in SQLite.
- Implemented event retrieval, entity/topic exploration, a lightweight local API/dashboard, and a research workflow that generates report-ready analyses from stored event data.

## Three-Bullet Version

- Built a local AI intelligence workbench that collects high-signal AI news from trusted domains and converts raw updates into structured events with topics, entities, confidence, and impact notes.
- Added historical search, event queries, entity timelines, lightweight APIs, and a local dashboard for browsing events, topics, entities, and saved research reports.
- Implemented a research workflow that generates Markdown reports from persisted events, supports optional live enrichment, and exposes report generation through both CLI and API entry points.

## Interview Framing

- Problem: raw AI news feeds are noisy, hard to audit, and difficult to turn into reusable research output.
- Approach: separate discovery, event extraction, persistence, retrieval, and research generation through explicit intermediate contracts and SQLite-backed state.
- Outcome: one system can support daily digests, historical search, event exploration, and research-style report generation without depending on external hosted infrastructure.

## Skills Demonstrated

- LLM application engineering
- event extraction and schema design
- retrieval and local knowledge system design
- API and lightweight product delivery
- research workflow orchestration
- test-driven iterative development
